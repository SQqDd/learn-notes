- 引用计数算法优缺点
  - 优点
    - 发现垃圾时立即回收
    - 最大限度减少程序暂停；对象的回收不需要另外的 GC 线程专门去做，业务线程自己搞定；内存空间爆满的时候，会立即执行去回收垃圾
  - 缺点
    - 无法回收循环引用的对象
    - 时间开销大；每次赋值操作的时候都要做相当大的计算，尤其这里面还有递归调用

```js
function fn() {
  const obj1 = {}
  const obj2 = {}

  obj1.name = obj2
  obj2.name = obj1
}

fn()

// fn 执行完的时候，obj1 和 obj2 是局部变量，理应要被回收掉（这两个数据空间没有任何作用了）
// 由于 obj1 引用了 obj2，obj2 引用了 obj1
// obj1 和 obj2 相互引用
// 此时他们的引用计数为 1
// 这就导致无法对这两个空间进行回收

```
